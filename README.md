---

## GitHub Pages Setup (Do This Once)

1. Go to your repo: `github.com/lovelaceyrae11/lovelaceyrae.github.io`
2. Click **Settings** → **Pages**
3. Under Source: select `main` branch, `/ (root)` folder
4. Click **Save**
5. Your site goes live at: `https://lovelaceyrae11.github.io`

## Commit Messages — Copy and Paste These

When you upload or change the `index.html` file on GitHub, you'll be asked to write a "commit message." Here are the ones to use:

**First upload:**
```
feat: launch Castleberry Bloom homepage

Combined framework + agents landing page. Replaces castleberrybloom.com.
Includes: CGF framework, 8 agents, 10-Cent Heist investigation, 
Castleberry Equations, Substack + Cash App links.
```

**When you update text:**
```
content: update [section name] copy
```

**When you fix something:**
```
fix: [what you fixed]
```

**When you add a new section:**
```
feat: add [section name] section
```

**Example of a real commit message:**
```
content: update investigation PRR status for OLCC PRR-2009
```

## How to Upload index.html to GitHub (Step by Step)

1. Go to `github.com/lovelaceyrae11/lovelaceyrae.github.io`
2. Click **Add file** → **Upload files**
3. Drag `index.html` into the upload box
4. Scroll down to "Commit changes"
5. In the first box, paste the commit message above
6. Click **Commit changes**
7. Wait 2-3 minutes
8. Go to `https://lovelaceyrae11.github.io` — it's live

## If You Get Your Domain Back

If `castleberrybloom.com` becomes available again, you can point it to GitHub Pages for free:

1. Go to Settings → Pages in your GitHub repo
2. Under "Custom domain" enter `castleberrybloom.com`
3. At your domain registrar (wherever you bought the domain), add a CNAME record pointing to `lovelaceyrae11.github.io`

GitHub Pages hosting is always free regardless of whether you use a custom domain.
